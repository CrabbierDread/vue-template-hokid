## Механизм авторизации
```
Загрузка виджета
  -> Загрузка модуля client
    -> Проверяем наличие token (cookie, localeStorage)
      -> Если токен существует
        -> запуск метода login с передачей token
          -> Отчищаем store, исключая поле token
          -> запрос get /client с передачей токена / Генерация глобального события 'client:login:request'
            -> Данные пришли успешно(response)
              -> Записываем данные в store / Генерация событиря 'client:login:success'
            -> Ошибка (error) / Генерация события 'client:login:error'
              -> Если причина ощибки отсутствие интернета
                -> Повторить попытку логина
```
